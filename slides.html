Benefits

  • Functional
  • Pure (almost)
  • Declarative

Nix Language

  • Numbers 42

  • Strings "hello"

  • List [1 2 3]

  • Expressions 1+2+3

  • Attribute Sets { key = value; ... }

  • Let Bindings let x = ... in ...

  • Functions f x

pure = no side effects, no variables = declarative functional = first-class support lazy = expression is not evaluated until value is needed * Let-Expressions * Pattern-Matching not-general purpose language dynamically types = no type signatures/compile time

  myNumber   = 3;

Let-Binding

Factor out common expression:

{
  name = otherAttrSet.a.b.c;
  src  = otherAttrSet.a.b.d; 
  ...
}
let common = otherAttrSet.a.b;
in
  {
    name = common.c;
    src  = common.d; 
    ...
  }

Plain assignment is not an expression, so not supported on its own, just as let…

Functions

Typescript

const f = x => 2x
f(3)

Nix

f = x: 2*x        # definition
f 3               # usage

Math

f: ℝ ⟶ ℝ
f: x ↦ 2x

Functions

Currying

g: (ℝ,ℝ) ⟶ ℝ                      g: ℝ ⟶ (ℝ ⟶ ℝ)
g: (x,y) ↦ x+y                    g: x ↦ (y ↦ x+y)

Usage

g(3,4)                            g(3)(4)

Functions

Typescript

const g = (x,y) => x+y
g(3,4)

Nix

  g = x: y: x+y   # definition
  g 3 4           # usage
((g 3) 4)
